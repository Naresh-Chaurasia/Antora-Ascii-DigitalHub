== Integration with VS Code


#Weather | Crypto | Github#

---

* Visual Studio Code
* Continue Extension
* #Continue CLI#
** Think of it like this:
*** Your FastMCP server = Tool provider (exposes get_weather)
*** LLM (GPT) = Brain (decides when to use the tool)
*** Continue = Connector / Controller
*** Continue = AI agent runtime inside VS Code that enables LLMs to use your MCP tools.
* OpenAI models
* MCP (Model Context Protocol) server
* Python MCP server executed using `uv`

This setup allows an AI model to call external tools (like a weather API) through MCP.

---

*1. Architecture Overview*

The overall architecture works as follows:

----
VS Code / Terminal
↓
Continue Extension / Continue CLI
↓
OpenAI Model (GPT)
↓
MCP Protocol
↓
MCP Server (weather.py)
↓
External Tool / API
----

In this setup:

* The LLM decides when to call a tool.
* MCP provides the standard interface.
* The MCP server executes the tool.

*5. Continue Configuration for VS Code*

VS Code uses the TypeScript configuration file:

---

`~/.continue/config.ts`

Example configuration:

[source, typescript]
----
const config = {
  defaultModel: "gpt-4o-mini",

  models: [
    {
      title: "GPT-4o-mini",
      provider: "openai",
      model: "gpt-4o-mini",
      apiKey: process.env.OPENAI_API_KEY
    }
  ],

  mcpServers: {
    weather: {
      command: "uv",
      args: [
        "run",
        "--directory",
        "/Users/nareshchaurasia/nc/Python-TechHub/AI-ML-GenAI/MCP/MCP-Henry-Habib/My-Code/hello-world",
        "mcp",
        "run",
        "weather.py"
      ]
    },

    crypto: {
      command: "uv",
      args: [
        "run",
        "--directory",
        "/Users/nareshchaurasia/nc/Python-TechHub/AI-ML-GenAI/MCP/MCP-Henry-Habib/My-Code/API",
        "mcp",
        "run",
        "crypto.py"
      ]
    },
    github: {
      command: "uv",
      args: [
        "run",
        "--directory",
        "/Users/nareshchaurasia/nc/Python-TechHub/AI-ML-GenAI/MCP/MCP-Henry-Habib/My-Code/API",
        "mcp",
        "run",
        "github.py"
      ]
    }
  }
};

export default config;
----

##################################################

Notes:

* `defaultModel` defines which model Continue will use.
* `mcpServers` registers available MCP tools.

== What is `continue`


.Continue Plugin
[%collapsible]
====
*Continue* is an AI coding assistant for developers that runs inside editors like Visual Studio Code and in the terminal.

* It connects **LLMs (like GPT, Claude, etc.)** with your **codebase and tools**.
* It lets you **chat with AI about your code**, generate code, refactor, or debug.
* It supports **tool calling using MCP (Model Context Protocol)** so AI can execute external tools (APIs, databases, scripts).
* It works with multiple models (OpenAI, Anthropic, local models, etc.).
* It also provides a **CLI (`cn`)** to run AI workflows from the terminal.

In simple terms:

*Continue = AI layer that connects your editor + LLM + tools.*

Typical flow:

```
Editor / Terminal
      ↓
Continue
      ↓
LLM (GPT / Claude)
      ↓
Tools via MCP
      ↓
Your code / APIs / databases
```

So Continue acts like a **bridge between your development environment and AI models**.
