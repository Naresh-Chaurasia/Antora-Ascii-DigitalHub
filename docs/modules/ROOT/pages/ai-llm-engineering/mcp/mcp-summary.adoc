== AI Dev Assistant for Backend Microservices - Detailed Plan

##################################################

*Project Goal*

* Build an MCP-based AI assistant focused on *backend microservices ecosystems*
* Help developers:
** Debug distributed services
** Analyze configs across services
** Monitor APIs and health
** Trace failures across microservices

* Outcome:
** MCP server with microservices-aware tools
** Integration with Spring Boot services
** Real-world debugging scenarios

##################################################

*High-Level Architecture*

* Components:
** MCP Server (Python - FastMCP)
** Tool Layer (Microservices tools)
** LLM Client (Visual Studio Code / GPT)
** Backend System (Spring Boot microservices)

* Microservices Setup:
** Service A (User Service)
** Service B (Order Service)
** Service C (API Gateway)
** Config Server (optional)
** Service Registry (Eureka/Consul - optional)

* Flow:
** User query → LLM (VS Code)
** LLM → MCP Tool
** Tool interacts with microservice (API/config/log)
** Response → LLM → User

=== 🛠 Phase 2: MCP Tools (Microservices-Focused)

*1. Service Config Analyzer*

* Function: `get_service_config(service_name)`

* Features:
** Read config per service
** Compare configs across services

* Use Cases:
** "What is DB config for user-service?"
** "Compare ports of all services"

##################################################


*2. API Health & Status Checker*

* Function: `check_service_health(service_name)`

* Features:
** Call actuator endpoints
** Aggregate health across services

* Use Cases:
** "Which services are down?"
** "Check order-service health"

##################################################

*3. Distributed Log Analyzer*

* Function: `search_logs(service_name, keyword)`

* Features:
** Read logs per service
** Correlate errors across services

* Use Cases:
** "Find errors in order-service"
** "Trace failure across services"

##################################################

*4. Service Dependency Mapper*

* Function: `get_service_dependencies(service_name)`

* Features:
** Identify downstream calls
** Map service interactions

* Use Cases:
** "Which service calls order-service?"
** "Show dependency chain"

##################################################

*5. Request Trace Tool (Advanced)*

* Function: `trace_request(trace_id)`

* Features:
** Track request across services
** Correlate logs using trace ID

* Use Cases:
** "Trace this failed request"
** "Where did it break?"


##################################################

*6. Code + Config Explainer*

* Function: `explain_service(service_name)`

* Features:
** Explain service purpose
** Analyze configs + code

* Use Cases:
** "Explain user-service"
** "What does this service do?"
