== Section 1

=== Why does MCP exists?

* Why **Model Context Protocol (MCP)** exists
** MCP addresses limitations of Large Language Models (LLMs) when interacting with external systems

* Limitation of LLMs
** LLMs are strong at generating content and providing conversational interfaces
*** Examples include writing emails, drafting marketing campaigns, and generating text
** LLMs cannot directly perform real-world actions
*** Sending emails
*** Running advertisements
*** Interacting with external services

* Tool and Function Calling as a Solution
** Tool calling allows LLMs and agents to interact with external systems
** An **agent** can be defined as an LLM combined with a set of tools

* Lack of Standardization in Tool Calling
** Different frameworks implement tool calling differently
*** OpenAI Agents Framework uses strict Python function-based tools
*** LangChain uses a different mechanism
** No universal standard exists for how LLMs call external tools

* Duplicate Development of Tool Integrations
** Developers repeatedly build the same integrations independently
*** Example: many developers build their own interface to the Gmail API
** This leads to duplicated effort across the industry

* Maintenance Problems with Custom Integrations
** External APIs change over time
*** Example: Gmail API updates can break existing integrations
** Developers must constantly maintain and update their custom tool interfaces

* Limitation of Server-Based Tool Calling
** Most existing tool calls rely on remote APIs and servers
*** Examples include Gmail API, Weather APIs, Airbnb APIs, and web services
** This prevents LLMs from easily interacting with a user's local environment

* Opportunities for Local Machine Integration
** Many useful AI interactions involve local machine resources
*** Organizing local files
*** Monitoring screen activity to suggest productivity improvements
*** Reading local source code in real time in editors like VS Code

* #Core Purpose of MCP#
** Provide a standardized way for AI systems to interact with external tools and systems
** Enable interoperability between different AI models and external services

* Key Benefit of MCP Standardization
** Developers can create MCP servers that expose tools and resources
** Any MCP client can connect to any MCP server
** This creates a reusable and interoperable ecosystem for AI integrations

* Nature of MCP
** MCP is not a revolutionary technology
** It is a widely adopted **standard protocol** that simplifies AI integrations

== Glossary

* Cline VS Code Plugin

=== What is `continue`


.Continue Plugin
[%collapsible]
====
*Continue* is an AI coding assistant for developers that runs inside editors like Visual Studio Code and in the terminal.

* It connects **LLMs (like GPT, Claude, etc.)** with your **codebase and tools**.
* It lets you **chat with AI about your code**, generate code, refactor, or debug.
* It supports **tool calling using MCP (Model Context Protocol)** so AI can execute external tools (APIs, databases, scripts).
* It works with multiple models (OpenAI, Anthropic, local models, etc.).
* It also provides a **CLI (`cn`)** to run AI workflows from the terminal.

In simple terms:

*Continue = AI layer that connects your editor + LLM + tools.*

Typical flow:

```
Editor / Terminal
      ↓
Continue
      ↓
LLM (GPT / Claude)
      ↓
Tools via MCP
      ↓
Your code / APIs / databases
```

So Continue acts like a **bridge between your development environment and AI models**.
