= LLM-Powered Journaling System (AsciiDoc)

== Overview

This system is designed to combine:
- Low-friction daily writing
- Local-first data storage
- LLM-powered reflection
- Long-term searchability

Goal:
Turn journaling into a *self-coaching and decision-making system*

---

== 1. Folder Structure

[source]
----
/journal/
  2026/
    04/
      2026-04-20.adoc
----

Optional:
[source]
----
/journal/reviews/
  2026-W16.adoc
----

---

== 2. Daily Journal Template

[source]
----
= 2026-04-20

== Raw Thoughts
Write freely. No structure.

== Events
-

== Decisions / Insights
-

== Emotional State
-

== Tags
#career #dance #learning

== LLM Reflection
// Fill after LLM analysis
----

---

== 3. Daily Workflow

*Step 1: Write*
- Open today's `.adoc` file
- Write for 5–10 minutes
- Focus on raw, honest thoughts

*Step 2: Extract Insight*
- Add at least 1 bullet in *Decisions / Insights*

*Step 3 (Optional): LLM Reflection*
- Copy content into LLM tool (Spidoc / Endora / local LLM)

---

== 4. LLM Prompt Template

[source]
----
Analyze this journal entry:

1. Summarize key points
2. Identify emotional patterns
3. Detect cognitive biases (if any)
4. Suggest 2 actionable improvements
5. Extract long-term themes
----

== 6. Tagging Strategy

Use consistent tags for searchability:

Examples:
- #mood/anxiety
- #confidence
- #career
- #dance
- #learning/genai

Benefits:
- Easy grep/search
- Pattern tracking over time

Future enhancements:

Example queries:
- "When did I feel similar emotions?"

---

== 10. Rules for Consistency

- Write daily (5–10 minutes)
- Don’t over-structure
- Don’t depend on LLM for writing
- Use LLM only for reflection
- Do weekly review (non-negotiable)

---

== 11. Key Principle

Consistency > Tools

This system works only if:
- You write regularly
- You reflect weekly

Avoid:
- Over-engineering
- Tool switching
- Complex workflows

---

== Final Outcome

If followed consistently, this becomes:

- Personal thinking system
